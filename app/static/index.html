What We've Implemented

Brocode's end-to-end electricity forecasting pipeline for 370 Portuguese clients (UCI Electricity Load Dataset, 2011–2014). All components are production-ready and fully deployed.

⚡ K-Means Client Clustering

We clustered all 370 electricity clients into 4 behavioral segments using temporal load features — mean hourly consumption, peak load hour, night/business/weekend load share, and load variability. Cluster assignment is deterministic and powers our entire model routing system.

🤖 Multi-Model Comparison

Seven model families evaluated head-to-head for every client: SARIMA, Prophet, LSTM, DeepAR, Temporal Fusion Transformer (TFT), XGBoost, and LightGBM with Optuna hyperparameter tuning. All models trained on strict chronological splits — zero data leakage.

🏆 Per-Client Best Model Selection

Every client is assigned its optimal model via MAPE on the held-out December 2014 test set (744 hours). XGBoost-Optuna wins for 335 out of 370 clients (90.5%), with the remainder served by Naïve Lag-24h, XGBoost, or LightGBM depending on load pattern.

🌤 Weather Feature Integration

External weather signals — temperature, relative humidity, and wind speed — sourced via the Open-Meteo API for all client regions across Portugal. Features are computed as hourly regressors and included in gradient-boosted models and Prophet, materially improving forecast accuracy for weather-sensitive clients.

💶 OMIE Daily Price Forecasting

We forecast daily spot-market electricity prices (EUR/kWh) from the Spanish/Portuguese OMIE exchange using Prophet with Fourier-mode yearly and weekly seasonality. Uncertainty bounds (85 %/115 % envelope) provide operational planning context for energy traders and grid operators — this is the next-7-day chart you see above.

📊 Quantile Forecasting with DeepAR

DeepAR (Amazon's autoregressive deep-learning model via GluonTS) provides probabilistic forecasts with P10/P50/P90 quantile outputs for clients with high load volatility. Uncertainty bounds surface alongside the point forecast, enabling risk-aware dispatch decisions and reserve capacity planning.

🔀 Cluster-Based Model Routing

A lightweight routing table maps each of 370 clients to their cluster's best model, enabling sub-millisecond forecast retrieval at query time. Model selection is fully auditable via the per-client MAPE leaderboard and routing table exported from the pipeline.

📅 Chronological Train/Test Splits

All models use a strict forward-in-time split — approximately 80 % training, 20 % validation, with the final December 2014 month (744 hours) held out as the test set across all clients. This prevents any look-ahead bias and ensures reported accuracy numbers reflect true out-of-sample performance.
